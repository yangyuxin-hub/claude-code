03 / 07

不是一问一答——
是循环执行

①
接收任务 → 分解成子步骤 LLM 自己决定下一步做什么
②
调用工具执行（读文件、写代码、运行命令） 工具结果返回给 LLM 继续判断
③
直到 LLM 判断「任务完成」 源码里的 while 循环就是这个逻辑

FINDING 03

提示词
工程
真相

Anthropic 的 System Prompt 比你想象的更「工程化」。

05 / 07
「不是几行 prompt，是精心设计的指令体系」——包含角色定义、行为约束、安全护栏、格式规范……
A
有专门的「拒绝策略」代码 哪些操作必须问用户确认，哪些直接拒绝
B
Context window 管理是关键 源码有完整的 token 压缩和对话剪枝逻辑
C
每个工具调用都有 retry 和 fallback 真正的工程健壮性，不是「祈祷 LLM 不出错」
