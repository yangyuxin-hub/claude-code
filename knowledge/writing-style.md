# 写作风格指南（持续迭代）

> 上次更新：2026-04-09
> 用途：每次写文章前读一遍，写完后对照检查，发布后根据反馈更新。
> 卡兹克写作风格执行层：直接用 `khazix-writer` skill。

---

## 受众假设

**目标读者**：也在研究 AI / agent / Claude 的人，有一定技术背景或主动学习的人。

**可以假设读者知道**：Claude 是什么、Claude Code 是什么、agent 的基本概念。

**写作密度**：直接用技术术语，不配生活化比喻；冲击感来自认知层面，不是扫盲感。

---

## 「小羊肖恩」差异化定位

**我不照搬卡兹克的**：他偏「创造力/存在感/人生意义」，我保留**商业变现视角**——「这对你的钱包/时间/竞争力意味着什么」是我的差异点，不能丢。

**我的素材库**：Claude Code 重度用户经历、AI 应用开发实践、给企业做自动化的经历。

**AI 辅助的边界**：我定判断和角度，AI 做执行和扩充。框架/方向全部自己定好再让 AI 填肉。

---

## 写作原则

**不做的事**：
- 不做资讯搬运（别人写过的事件不重复，只写我有不同判断的）
- 不做泛泛预测（「AI 要改变世界」这种话没有意义）
- 不做纯框架文（没有故事/案例支撑的结构拆解是废话）

**结构**：先说事件 → 讲表层逻辑 → 拆深层逻辑 → 落到「对我们意味着什么」

---

## 发布前自检

- [ ] 开头第一句是「现象/场景」，不是「结论/观点」？
- [ ] 有没有先共情读者感受，再给出判断？
- [ ] 标题里有没有具体数字或具体场景？
- [ ] 开头 100 字能不能让人读下去？
- [ ] 有没有至少一个「我之前不知道」的信息点？
- [ ] **活人感终审**：读完这篇，像有见识的普通人在聊，还是 AI 在输出信息？
- [ ] 一个没关注我的陌生人看到这篇，会转发吗？
