# 个人定位与商业方向

## 身份

Claude Code 深度用户，帮企业/老板落地 AI 自动化。

## 核心能力（已验证）

- 公众号内容自动发布
- 论文/文档批量整理

## 目标客户画像

**行业**：金融（基金/券商/保险）、电商（品牌方/MCN）

**筛选标准**：
- 钱多，付费意愿强
- 信息化程度高（有系统有数据，容易对接）
- 有固定 SOP（结构化工作流 = AI 最容易替代的部分）
- 业务可复制（做一个客户等于做一类客户，边际成本低）

## 金融行业切入场景

- 研报 / 日报自动生成
- 客户报告批量生产
- 合规文件整理归档
- 会议纪要 → 结构化输出

## 电商行业切入场景

- 商品描述批量生成
- 客服话术自动回复
- 数据报表自动汇总
- 竞品信息监控整理

**现在**：AI 认知科普，受众偏泛

**应该**：让行业老板看完就想找你

> 错误示范：《AI 海啸来了，你准备好了吗？》
>
> 正确示范：《一个基金公司用 AI 每天省了 3 小时研报时间》
> 《电商运营用 Claude Code 批量生产详情页，成本降了 80%》

核心逻辑：**具体场景 + 具体数字 + 可复制**

## 案例库策略

### 为什么案例文档是核心资产

一份案例文档同时服务三个用途：

1. **公众号文章** → 吸引潜在客户
2. **AI 上下文** → 下次做类似项目直接调用经验
3. **销售素材** → 见客户时的信任背书

### 深挖路径

- **横向**：把一个场景拆成多个子场景（如公众号发布 → 图文/视频号/定时发布）
- **纵向**：找 3-5 个同行业客户验证，每个写一篇
- **提炼**：积累 10 个案例后，总结「AI 自动化 SOP 方法论」

方法论出来后可以：卖课 / 卖模板 / 卖咨询

### 案例文档标准格式

见 `cases/` 目录，每个案例包含：

```
客户背景    → 行业/规模/原来怎么做/痛点
解决方案    → 工具/流程/关键技术点
效果        → 省了多少时间/降了多少成本（具体数字）
可复制性    → 哪些行业/场景可以直接套用
经验教训    → 踩过的坑/下次注意的点
```
